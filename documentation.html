Overview

AMAR Engine (AME) is a reality mapping and physical reconstruction system, not a traditional 3D rendering engine. It "forcefully" instantiates entities with physical authority in the digital dimension through resampling of high-dimensional spatial point clouds and integration of multimodal semantic injection.

Core Mission

PHYSICAL INSTANTIATION OF REALITY

System Vision

In AME's worldview, everything can be Instantiated, and physical laws are absolutely governed by the underlying engine AMAR GENIS. The AME system aims to create a digital ecosystem that can truly reflect and simulate the real world, where each entity has physical properties and behaviors highly consistent with the real world.

System Components

AME Scanner

Spatial skeleton capture with density collapse processing and OBB fitting, providing geometric constraints for entities. It serves as the executor of the 3DGS path.

AMAR ENGINE

System core with Normalization Center (Admin), integrating metaclass, AEID, and skills systems. It acts as the system hub, collapsing data from three paths into a unique physical entity.

AMAR GENIS

Physics engine and executor of physical laws, implementing physical instantiation and behavior simulation. It serves as the underlying physics engine and physical judge of the AME world.

Architecture

Triple-Path Convergence

The AME system adopts a triple-path convergence architecture, with Admin as the hub, collapsing data from three heterogeneous paths into a single physical entity: 3DGS Path, Media Input Path, and LLM API Path.

3DGS Path

Navigates 3D Gaussian splatting through density-weighted sampling algorithms to extract spatial truth, providing geometric constraints for entities.

Media Input Path

Processes cross-spectrum images, temporal videos, and acoustic data to determine object surface material flow and acoustic impedance, providing sensory consistency for entities.

LLM API Path

Converts natural language fragments into metaclass descriptions, injecting physical constants, and providing causality for entities.

Core Concepts

Metaclass System

Genetic genes of the system, with evolutionary logic from concrete to abstract to self-consistent. Includes Root Metaclass (axioms of the physical world) and Standard Metaclass (entity templates with interaction potential).

AEID System

Advanced Entity Identification system that uniquely identifies and tracks entities across different domains, ensuring consistent identification throughout the system.

Skills System

A modular system for adding and managing capabilities and behaviors to entities, allowing for flexible extension of entity functionality.

Physical Handle Binding

Implements asynchronous coupling between the 3DGS visual layer and physical collision layer, supporting topological mutations and ensuring consistency between visual presentation and physical behavior.

API Reference

Entities API

POST /api/entities: Create entity with GSOutput + MediaData + SemanticData
GET /api/entities/{id}: Get entity information
PUT /api/entities/{id}: Update entity properties

Metaclasses API

GET /api/metaclasses: Get metaclass list
POST /api/metaclasses: Create new metaclass
