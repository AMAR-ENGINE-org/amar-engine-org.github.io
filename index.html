About AMAR Engine

AMAR Engine (AME) is a next-generation 3D tool designed to generate virtual worlds from both natural language descriptions and real-world captures. It serves as a decentralized operating system for virtual reality where the physical behavior of objects is governed by a unique Metaclass system.

Core Philosophy

  • Real Mesher, Not Just Models: Unlike NeRF or 3DGS, AME generates true geometric meshes. This ensures that every asset is a "solid" entity compatible with standard 3D environments.
  • Metaclass Logic: Properties are not inherited but composed. A Metaclass defines what an object is and what it can do (e.g., a "Container" Metaclass allows an object to hold liquid).
  • Decentralized Collaboration: AME is a local tool. It uses Cloud-based LLMs to guide the generation process via the Skills Library, overcoming local hardware limitations.
  • Global Identity (AEID): Every scene and asset is assigned a worldwide unique ID (AEID) for registration and tracking in a global database.

Technical Architecture

The Asset vs. The World

Assets: Individual entities (like a teacup) with specific Metaclass attributes.

Scenes: Entire worlds composed of multiple assets. All scenes are fully compatible and can be combined like meshes in Unreal Engine.

Generation Pipeline

Instruction (Skills Library): A RESTful-style specification that allows an AI to guide AME on how to build a scene or asset.

Reconstruction: Local AME processes camera footage (photos/videos) or generative prompts to output high-quality Meshes.

Attribution: Metaclass schemas are injected into the Mesh to define physical interactions without traditional class inheritance.

Triple-Path Convergence Architecture

3DGS Path [AME Scanner]: Spatial skeleton capture with density collapse processing and OBB fitting.

Media Input Path [Media Loader]: Multimodal attribute sourcing from cross-spectrum images, temporal videos, and acoustic data.

LLM API Path [Semantic Coder]: Metaclass definition and logic assignment, converting natural language fragments into Metaclass descriptions.

Key Features

Metaclass System

The genetic genes of the AME system, defining physical properties and behavioral rules for entities. Includes Root Metaclasses (physical world axioms) and Standard Metaclasses (entity templates with interaction potential).

AEID System

AMAR Engine ID system that assigns worldwide unique identifiers to every scene and asset for registration and tracking in a global database.

Skills System

The "API of the Virtual World" - a standardized way for AI to communicate construction intent to AME, focusing on the "what" and "how" of an object's properties.

Mesher System

Generates true geometric meshes (not just models) ensuring every asset is a "solid" entity compatible with standard 3D environments.

AME Admin

The system's central hub (Normalization Center) that collapses data from three heterogeneous paths into a single physical entity.

AMAR GENIS

The underlying physics engine that enforces physical laws, implements real-time metaclass injection, and supports topological mutation.

System Components

AME Scanner

3DGS path executor responsible for spatial skeleton capture, density collapse processing, and OBB fitting. Provides geometric constraints for entities.

Media Input Path

Handles multimodal attribute sourcing from cross-spectrum images, temporal videos, and acoustic data. Provides perceptual consistency for entities.

LLM API Path

Responsible for metaclass definition and logic assignment, converting natural language fragments into Metaclass descriptions and injecting physical constants.

AME Admin

The central hub (Normalization Center) that collapses data from three heterogeneous paths into a single physical entity. Performs metaclass inference and parameter normalization.

AMAR WORLD

The persistent structured output of the system, a directory-based world format that is metaclass-resolved, parameter-normalized, and dependency-explicit.

AMAR GENIS

The underlying physics engine that enforces physical laws, implements real-time metaclass injection, and supports topological mutation.

Technical Innovations

Triple-Path Convergence Architecture

Collapses data from three heterogeneous paths (3DGS, Media Input, LLM API) into a single physical entity through the Admin normalization center.

Metaclass Protocol

Evolutionary logic from concrete to abstract to self-consistent, with Root Metaclasses as physical world axioms and Standard Metaclasses as entity templates.

Physical Handle Binding

Implements asynchronous coupling between the 3DGS visual layer and physical collision layer for seamless interaction.

Real-time Metaclass Injection

Enables real-time changes to an object's physical constants without restarting the simulation, allowing for dynamic behavior adjustments.

Topological Mutation Support

Real-time collision body reconstruction algorithm designed for 3DGS dynamic flows, enabling fluid and dynamic object interactions.

Skills Specification

The "API of the Virtual World" - a RESTful-style specification that allows AI to guide AME on how to build scenes and assets.
